## Overview

AgroSentry is an IoT-based crop field management system originally built on the ARM7 microarchitecture (see the [hardware project](https://github.com/Shayantika-Dhar/AgroSentry-SmartCrop-Guardian)). This repository adds a full **data analytics layer** on top — simulating the sensor readings the hardware would produce and running a complete analysis pipeline from raw CSV through SQL to an interactive web dashboard.

**Dataset:** 12,000+ sensor readings across 3 crop fields and 4 sensor nodes, sampled hourly over 1,000 hours.

**Live dashboard:** [shayantika-dhar.github.io/AgroSentry-SmartCrop-Guardian](https://shayantika-dhar.github.io/AgroSentry-SmartCrop-Guardian)

## What was analysed

| Analysis | Method | Output |
|---|---|---|
| Diurnal temperature cycles | pandas feature engineering + groupby | daily_summary.csv |
| Field-level health comparison | SQL GROUP BY + AVG aggregation | sql_field_averages.csv |
| Drought risk alerts (soil < 25%) | SQL CASE WHEN + threshold filter | sql_drought_alerts.csv |
| Sensor anomaly detection | 2σ statistical rule (pandas) | anomalies.csv |
| Peak heat hours (24h cycle) | SQL GROUP BY hour | sql_peak_temp_by_hour.csv |
| Sensor reliability ranking | SQL GROUP BY + anomaly rate % | sql_sensor_anomalies.csv |
| Soil stress risk scoring | SQL CASE WHEN multi-tier | sql_soil_stress_ranking.csv |
| Correlation analysis | pandas .corr() | printed in analyze.py |

## Key SQL techniques demonstrated

```sql
-- CASE WHEN: Stress level classification
SELECT date, field,
       CASE
           WHEN AVG(soil_moisture_pct) < 20 THEN 'Critical'
           WHEN AVG(soil_moisture_pct) < 30 THEN 'High'
           WHEN AVG(soil_moisture_pct) < 40 THEN 'Medium'
           ELSE 'Normal'
       END AS stress_level
FROM sensor_readings
GROUP BY date, field
HAVING stress_level IN ('Critical', 'High')
ORDER BY AVG(soil_moisture_pct) ASC;
```

Other SQL patterns used: `GROUP BY` + `HAVING`, `ROUND()`, `SUM(CASE WHEN ...)` for conditional counting, `ORDER BY`, subqueries, and date extraction with `strftime`.

## How to run locally

```bash
# 1. Clone the repo
git clone https://github.com/Shayantika-Dhar/AgroSentry-SmartCrop-Guardian.git
cd AgroSentry-SmartCrop-Guardian

# 2. Install dependencies
pip install pandas numpy

# 3. Generate sensor data
python simulate_data.py

# 4. Run pandas analysis + export dashboard JSON
python analyze.py

# 5. Run SQL analysis (optional — saves CSV results)
python sql_analysis.py

# 6. Serve the dashboard locally
python -m http.server 8000
# Open http://localhost:8000
```

## Key findings from the data

- **Field B** consistently runs ~1.5°C warmer than Field A, suggesting microclimatic variation
- **Soil moisture** follows a weekly stress cycle — dips on high-temperature days
- **Peak crime hours** for temperature anomalies: 13:00–15:00 (peak heat window)
- **3.2% of readings** were flagged as anomalous via 2σ rule — likely sensor calibration drift
- **Drought alerts** were most concentrated in Field B during the simulated dry periods
